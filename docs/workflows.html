GitHub Actions

✓ CI Pipeline ci.yml

The main quality gate. Runs on every push and pull request targeting main. Blocks merging if any step fails.

Trigger

EventBranchBehavior
pushmainFull quality check on latest commit
pull_requestmainQuality check on PR head commit

Environment Variables (Secrets)

SecretPurpose
NEXT_PUBLIC_SUPABASE_URLSupabase project URL for builds
NEXT_PUBLIC_SUPABASE_ANON_KEYSupabase anonymous key
SUPABASE_SERVICE_ROLE_KEYSupabase service role key (server-side)

Steps

  1. Checkout code — actions/checkout@v4 clones the repository
  2. Setup pnpm — pnpm/action-setup@v4 installs pnpm (version from packageManager field)
  3. Setup Node.js 20 — actions/setup-node@v4 with pnpm cache enabled
  4. Install dependencies — pnpm install --frozen-lockfile (deterministic install)
  5. Biome check — pnpm biome check . validates lint + formatting
  6. TypeScript type check — pnpm typecheck runs turbo typecheck across all packages
  7. Build all apps — pnpm build runs turbo build for all Next.js apps
  8. Unit tests — pnpm --filter @repo/utils test runs Vitest tests
  9. Dead code detection — pnpm knip finds unused exports and dependencies
  10. Cache Turborepo — actions/cache@v4 caches .turbo directory keyed by OS + SHA

Workflow YAML

# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

env:
  NEXT_PUBLIC_SUPABASE_URL: ${{ secrets.NEXT_PUBLIC_SUPABASE_URL }}
  NEXT_PUBLIC_SUPABASE_ANON_KEY: ${{ secrets.NEXT_PUBLIC_SUPABASE_ANON_KEY }}
  SUPABASE_SERVICE_ROLE_KEY: ${{ secrets.SUPABASE_SERVICE_ROLE_KEY }}

jobs:
  quality:
    name: Quality checks
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm
      - name: Install dependencies
        run: pnpm install --frozen-lockfile
      - name: Biome check (lint + format)
        run: pnpm biome check .
      - name: TypeScript type check
        run: pnpm typecheck
      - name: Build all apps
        run: pnpm build
      - name: Run unit tests
        run: pnpm --filter @repo/utils test
      - name: Dead code detection
        run: pnpm knip
      - name: Cache Turbo
        uses: actions/cache@v4
        with:
          path: .turbo
          key: turbo-${{ runner.os }}-${{ github.sha }}
          restore-keys: turbo-${{ runner.os }}-

Workflow YAML

# .github/workflows/labeler.yml
name: PR Labeler

on:
  pull_request_target:
    types: [opened, synchronize]

jobs:
  label:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      pull-requests: write
    steps:
      - uses: actions/labeler@v5
        with:
          repo-token: ${{ secrets.GITHUB_TOKEN }}
          configuration-path: .github/labeler-config.yml
GitHub Actions

⚡ Smoke Tests smoke.yml

End-to-end Playwright tests that verify all four apps work correctly in a browser environment. Runs automatically after CI passes on main, or can be triggered manually.

Trigger

EventCondition
workflow_dispatchManual trigger from GitHub UI
workflow_runAfter CI workflow completes on main with success conclusion

Steps

  1. Checkout code — actions/checkout@v4
  2. Setup pnpm — pnpm/action-setup@v4
  3. Setup Node.js 20 — actions/setup-node@v4 with pnpm cache
  4. Install dependencies — pnpm install --frozen-lockfile
  5. Install Playwright browsers — pnpm --filter e2e exec playwright install chromium
  6. Run smoke tests — pnpm smoke which runs pnpm --filter e2e test (Playwright)
  7. Upload artifacts on failure — uploads e2e/test-results/ screenshots as GitHub artifacts

Playwright Projects

The e2e test suite covers four apps via separate Playwright projects:

  • web — Public storefront
  • admin — Admin dashboard
  • staff — Staff order management
  • rider — Rider delivery app

Workflow YAML

# .github/workflows/smoke.yml
name: Smoke Tests

on:
  workflow_dispatch:
  workflow_run:
    workflows: [CI]
    branches: [main]
    types: [completed]

jobs:
  smoke:
    if: ${{ github.event_name == 'workflow_dispatch' || github.event.workflow_run.conclusion == 'success' }}
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm
      - name: Install dependencies
        run: pnpm install --frozen-lockfile
      - name: Install Playwright browsers
        run: pnpm --filter e2e exec playwright install chromium
      - name: Run smoke tests
        run: pnpm smoke
        env:
          NEXT_PUBLIC_SUPABASE_URL: ${{ secrets.NEXT_PUBLIC_SUPABASE_URL }}
          NEXT_PUBLIC_SUPABASE_ANON_KEY: ${{ secrets.NEXT_PUBLIC_SUPABASE_ANON_KEY }}
          SUPABASE_SERVICE_ROLE_KEY: ${{ secrets.SUPABASE_SERVICE_ROLE_KEY }}
      - uses: actions/upload-artifact@v4
        if: failure()
        with:
          name: playwright-screenshots
          path: e2e/test-results/
Local Developer Hooks

🔒 Pre-commit Hooks Parallel

Runs three quality checks in parallel on every local git commit via Husky → Lefthook. Catches issues before code is pushed.

Hook Chain

git commit
  → .husky/pre-commit (→ npx lefthook run pre-commit)
    → Lefthook runs 3 commands in parallel:
       → format-lint — Biome auto-fix on staged files
       → typecheck — Turborepo typecheck on affected packages
       → secrets — TruffleHog scans last commit for secrets

Commands

CommandWhat it doesFilter
format-lint Runs pnpm biome check --write --no-errors-on-unmatched {staged_files} — auto-fixes lint/format on staged files *.{js,ts,jsx,tsx,json,md}
typecheck Runs pnpm turbo typecheck --affected — type checks only packages changed since main All staged files
secrets Runs npx trufflehog git --since=HEAD~1 --no-verification --fail — scans for leaked API keys/secrets Last commit
Note: The secrets command includes 2>/dev/null || true as a fallback, so it won't block commits if TruffleHog encounters non-critical issues. Only actual secret matches will fail the hook.

Configuration Files

lefthook.yml

pre-commit:
  parallel: true
  commands:
    format-lint:
      run: pnpm biome check --write --no-errors-on-unmatched {staged_files}
      glob: "*.{js,ts,jsx,tsx,json,md}"
    typecheck:
      run: pnpm turbo typecheck --affected
    secrets:
      run: npx trufflehog git --since=HEAD~1 --no-verification --fail 2>/dev/null || true

.husky/pre-commit

#!/usr/bin/env sh
. "$(dirname -- "$0")/_/husky.sh"

npx lefthook run pre-commit

Setup: pnpm prepare installs Husky git hooks, which delegates to Lefthook.

Build Orchestrator

⚡ Turborepo Pipeline Cached

Turborepo orchestrates all build, typecheck, and dev tasks across the monorepo with intelligent caching and dependency-aware parallelism.

Task Definitions

TaskDepends OnOutputsCached
build ^build (dependencies first) .next/**, dist/** Yes
dev None None (persistent) No
lint ^build None Yes
typecheck ^build None Yes
clean None None No
Global env vars: Turborepo tracks NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY, SUPABASE_SERVICE_ROLE_KEY, and DATABASE_URL as global cache keys. Changing any of these invalidates the cache.

Full Configuration

// turbo.json
{
  "$schema": "https://v2-10-2.turborepo.dev/schema.json",
  "globalDependencies": ["**/.env.*local"],
  "globalEnv": [
    "NEXT_PUBLIC_SUPABASE_URL",
    "NEXT_PUBLIC_SUPABASE_ANON_KEY",
    "SUPABASE_SERVICE_ROLE_KEY",
    "DATABASE_URL"
  ],
  "tasks": {
    "build": {
      "dependsOn": ["^build"],
      "outputs": [".next/**", "!.next/cache/**", "dist/**"]
    },
    "dev": { "cache": false, "persistent": true },
    "lint": { "dependsOn": ["^build"] },
    "typecheck": {
      "dependsOn": ["^build"],
      "outputs": [],
      "inputs": ["$TURBO_DEFAULT$", "tsconfig.json"]
    },
    "clean": { "cache": false }
  }
}
